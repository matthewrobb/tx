# v4-refactor Research

## Source

`.twisted/todo/v4-redesign/SPEC.md` — complete design specification, dated 2026-04-01.

## Summary

twisted-workflow v4 is a ground-up redesign built on three core insights from v3:

1. **Artifact existence is the only reliable progress signal** — boolean flags in state.json drift; files either exist or they don't. The v4 engine derives all step completion from artifact presence (+ predicate queries on artifact content).
2. **The pipeline should be data, not code** — v3 hardcoded step sequences in TypeScript. v4 externalizes the entire pipeline into `settings.json`: lanes, steps, artifact deps, type routing. The TypeScript engine becomes a generic config interpreter.
3. **Work model needs a third tier** — v3 had epics + tasks. v4 adds stories between them: epic → story (user-facing value) → task (technical unit).

## Key Changes from v3

### Terminology
- "objective" → "epic" everywhere
- `-o`/`--objective` flag → `-e`/`--epic` flag

### Lane Model (filesystem-mirrored, numeric-prefix)
- `0-backlog` → `1-ready` → `2-scoped` → `3-started` → `4-done` → `5-archived`
- Lane advancement is automatic via `tx next` (artifact-driven)
- Two exceptions require human decision: `0-backlog → 1-ready` (`tx ready`) and archiving (`tx archive`)

### Epic Types (5 types with distinct pipelines)
- `feature`: full pipeline, stories + estimation
- `technical`: full pipeline, no stories, effort only
- `bug`: abbreviated (skips `2-scoped`)
- `spike`: research pipeline, promote-or-archive outcome
- `chore`: minimal pipeline, no estimation

### Spike Promotion
- `tx promote <spike> --type feature` mutates in place — changes type, advances lane/step pointers, carries artifacts forward
- Artifact-driven skip logic auto-skips steps whose artifacts already exist

### Stories (`feature` + `technical` only)
- `stories.json` produced by new `decompose` step in `2-scoped`
- Story schema: id (S-001), summary, acceptance_criteria, status, story_points, tasks[]

### Estimation (two dimensions)
- `story_points`: Fibonacci (1/2/3/5/8/13) — features only, value + uncertainty
- `effort`: XS/S/M/L/XL — features, technical, bugs; technical complexity
- `timebox`: minutes — spikes only
- `confidence`: low/medium/high — all types
- Estimation happens in `estimate` step in `1-ready`, produces `estimate.json`

### Retrospective Loop
- Task agents write retro notes (`type: "retro"`) as they finish
- At `tx close`: all retro notes → `retro.md` → backlog candidates → new `0-backlog` epics
- Human triages candidates via `tx backlog promote`

### Configuration (v4 schema)
- Top-level: `version: 4`, `lanes`, `steps`, `types`, `estimation`, `commit`, `daemon`
- `StepConfig.produces[]` declares artifact outputs; `LaneStep.requires[]` declares artifact inputs
- `ArtifactRef` can be a plain path string or a `PredicateRef` with `where` clause
- Built-in predicates: `tasks.all_done`, `tasks.any_done`, `tasks.any_committed`, `tasks.all_committed`, `tasks.count >= N`
- Three-layer merge preserved: `deepMerge(defaults, ...presets, projectSettings)`

### Engine Algorithm (`tx next`)
- Pure function: given `(epicDir, config)`, evaluates steps in current lane
- Step status: `done` (all produces exist) | `runnable` (requires met) | `blocked` (requires unmet)
- Runs first runnable step → recurse; if all done → advance lane; if entry_requires unmet → blocked
- Recurses after lane advance (handles empty lanes, auto-skips artifact-satisfied steps)

### State Schema (v4 `CoreState`)
- Adds: `type`, `lane`, `disposition` (promoted/abandoned/deprecated), `promoted_to`, `estimate`
- `lane` is filesystem truth — directory location wins over `state.json` if they diverge
- `step` is informational; engine derives runnable step from artifacts, not this field
- No boolean completion flags (`research_done: true` etc.)

### `tx commit` (new command)
- Only supported way to create git commits
- Formats: `({epic}#{task}): {message}` (configurable)
- Writes short commit hash back to `tasks.json` for active task
- `tx init` installs `commit-msg` hook that rejects bare `git commit`
- Escape: `--no-task` for commits outside epic/task context

### Daemon Architecture
- Optional on-demand daemon via `sock-daemon` (by isaacs)
- One daemon per machine (project-aware), self-terminates after idle timeout (default 5 min)
- Writes `server.json` to `.twisted/` on startup
- Fallback: direct filesystem access with `proper-lockfile`

### Agent Files
- Live in `.twisted/agents/`, symlinked to `.claude/agents/twisted/` via `symlink-dir`
- Roles: build.md, planning.md, retro.md, review.md, research.md
- **Generated** by `npm run build` from config — not hand-edited

### XState v5 Integration
- Engine uses XState v5 for step sequencing and lane transitions
- Per-epic `epicMachine` with states: evaluating → running → advancingLane → blocked/complete/failed/archived
- Persisted snapshot = `state.json`; rehydrated on each `tx next` invocation
- XState does NOT own file I/O, skill invocation, config resolution, or output formatting

### Runtime Change
- **Bun dropped** — Node.js >= 20 required
- npm replaces bun as package manager
- Test runner: vitest (replaces bun test)

## New Dependencies
- `sock-daemon ^1.x` — daemon lifecycle
- `fs-extra ^11.x` — filesystem operations
- `symlink-dir ^6.x` — cross-platform symlinks
- `proper-lockfile ^4.x` — file locking fallback
- `xstate ^5.20.0` — state machine engine

## New CLI Commands
- `tx ready <epic>` — move from backlog to ready (human commit gate)
- `tx archive <epic> --reason` — explicit archival
- `tx promote <epic> --type` — spike promotion
- `tx estimate [epic]` — run estimate step
- `tx stories` — manage stories
- `tx commit "<msg>"` — tracked git commits
- `tx migrate` — v3→v4 migration (renames lanes, updates state.json, removes dead config)
- `tx backlog promote <note-id>` — promote retro candidate to new epic

## Migration from v3
- `tx migrate` command: non-destructive (creates `.twisted-v3-backup/` tarball)
- Renames lane dirs (`todo/` → `0-backlog/`, `in-progress/` → `3-started/`, etc.)
- Updates `state.json`: adds `type: "feature"` (default), `lane`, `metadata: {}`
- Removes dead config keys (see SPEC §18.2)
- v3 and v4 cannot coexist in same `.twisted/` — one-way migration

## Deferred (explicitly out of v4 scope)
- Planning poker, persona-based agent assignment, cross-epic dependencies, sprint containers, velocity tracking, web UI, multi-user mode
